## Add Backup Log to FieldLog PDFs (backend-only, temporary LA timezone)

### Decisions locked in
- **Timezone**: hard-coded `America/Los_Angeles` for now. Documented as TEMPORARY; will be replaced by client-supplied IANA timezone in next App Store update.
- **Sentinels**: excluded from Backup Log.
- **Frontend**: zero changes.
- **Schema**: zero changes.
- **AI report generation**: unchanged.

### Files

| Action | File | Purpose |
|---|---|---|
| Create | `supabase/functions/_shared/pdf.ts` | Multi-section `generatePdfBytes(sections[])`, `buildBackupLog(logs, tz?)`, `SENTINEL_TRANSCRIPTS`, `isSentinel`, `DEFAULT_BACKUP_TZ = "America/Los_Angeles"` |
| Edit | `supabase/functions/generate-report/index.ts` | Remove local `generatePdfBytes` + sentinels; import shared; build report + backup log sections; track usable logs alongside transcripts |
| Edit | `supabase/functions/regenerate-report-pdf/index.ts` | Remove local `generatePdfBytes`; import shared; fetch `voice_logs` via `daily_reports.log_ids`; build two-section PDF |
| Deploy | `generate-report`, `regenerate-report-pdf` | Push updated code |

### Multi-section PDF generator
```ts
generatePdfBytes([
  { content: reportContent },     // pages 1..N (main report)
  { content: backupLogContent },  // page N+1.. (Backup Log, fresh page)
])
```
Each section paginates independently → guaranteed hard page break between report and Backup Log. Footer "Generated by FieldLog" preserved on every page. Existing margins, fonts, line wrapping, escaping unchanged.

### Backup Log format
```
BACKUP LOG
Source voice notes — verbatim transcripts

9:14 AM — Voice Note
<verbatim transcript>

9:31 AM — Voice Note
<verbatim transcript>
```
- Sorted by `recorded_at` ascending.
- Excludes any log where `transcript` is null, empty, or in `SENTINEL_TRANSCRIPTS`.
- Times formatted via `toLocaleTimeString("en-US", { ..., timeZone: "America/Los_Angeles" })`.
- If zero usable logs: "No voice notes available." (still on its own page).

### `generate-report` integration
- Track `usableLogs: VoiceLogForBackup[]` in parallel with the existing `transcripts` array — push the log row at the same point a transcript is accepted (both in the "existing transcript" branch and the "freshly transcribed" branch).
- Empty-report path: `generatePdfBytes([{ content: emptyReport }, { content: buildBackupLog([]) }])` — Backup Log page will say "No voice notes available."
- Normal path: `generatePdfBytes([{ content: reportContent }, { content: buildBackupLog(usableLogs) }])`.
- Same `pdfPath` upload; same DB upserts; same response shape.

### `regenerate-report-pdf` integration
- After resolving `user_id`, look up the matching `daily_reports` row to read `log_ids`.
- `supabase.from("voice_logs").select("recorded_at, transcript").in("id", log_ids)` → pass to `buildBackupLog`.
- If lookup fails or returns nothing, Backup Log still renders ("No voice notes available.") — never blocks PDF regeneration.
- Generate two-section PDF; upload to same path; same DB updates.

### Preserved (unchanged)
- AI prompts and report content
- Footer text, fonts, margins, line wrapping
- Storage path: `${report_date}/${sanitizedName}_Daily Report_${report_date}.pdf`
- Email + download both read this path → both automatically include Backup Log
- All auth/JWT resolution logic
- All `daily_reports` / `admin_reports` upsert logic

### ⚠️ Temporary note (must be communicated)
Backup Log timestamps use `America/Los_Angeles` as a hard-coded default. **This must be replaced with the user's actual IANA timezone in the next App Store update**, by:
1. Mobile app sends `Intl.DateTimeFormat().resolvedOptions().timeZone` in the `generate-report` request body.
2. Edge function reads `body.timezone` and passes it to `buildBackupLog(logs, body.timezone)`.
3. `regenerate-report-pdf` does the same.

The shared module already accepts an optional `timezone` parameter, so the future update is a one-line change in each edge function plus the mobile request body.
